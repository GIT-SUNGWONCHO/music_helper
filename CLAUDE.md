# Music Helper — Claude 작업 기준

## 앱 목표

나 + 지인이 쓰는 기타 코드 악보 웹앱.

1. 곡 제목 입력 → AI가 가사 + 코드 생성
2. 악보 보면서 코드 직접 수정
3. 저장해서 계속 사용
4. 어디서나 내 악보 볼 수 있도록 (기기 간 동기화 — 미구현)

무료 지향. 복잡한 기능보다 이 네 가지가 잘 되는 게 우선.

## 작업 원칙

### 코드 건드리기 전에 반드시
- 구현 방향을 한 줄로 먼저 말하고 사용자 확인 후 진행
- "이렇게 할게요" → 사용자 OK → 구현
- **디자인(색/여백/버튼/인풋 등) 관련 작업은 새로 정하지 말고 먼저 `STYLE.md` 확인 후 있는 걸 재사용**

### 절대 하지 말 것
- 특정 곡/코드 이름을 일반 규칙의 예시로 박지 않기 (예: "Creep의 Bsus4" → 일반 규칙에 넣으면 안 됨)
- 목표에 없는 기능 추가하지 않기
- 목표 달성과 무관한 리팩토링, 정리 하지 않기
- 범위를 넘는 변경은 먼저 물어보기

### 방향 잃을 때 체크리스트
1. 이 변경이 위의 목표 1~4 중 어디에 해당하는가?
2. 해당하지 않으면 → 하지 않거나 먼저 물어보기

## 스택 요약

- React 18 + TypeScript + Vite (프론트엔드 전부)
- Dexie/IndexedDB (로컬 저장)
- Google Gemini API (AI 생성)
- 백엔드 없음 (현재)
